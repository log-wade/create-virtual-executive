# Cycle 11 — D2 Barge-in (TwiML) + outbound (admin API)

## Summary

- **Barge-in (Gather path):** `lib/twilio/voice-settings.ts` — `TWILIO_VOICE_BARGE_IN` (default on). When on, assistant reply + follow-up prompt are **nested inside one `<Gather input="speech">`** so callers can interrupt `<Say>` by speaking (Twilio speech barge-in). When off, legacy **Say then Gather** TwiML is used.
- **Persona on gather URLs:** `?personaId=` on all `/api/twilio/voice/gather` actions so **outbound** and **no-Redis** deployments still resolve the correct executive; **Redis `getCallPersona`** overrides when present.
- **Voice webhook:** Resolves persona from query (if valid) or env; stores `setCallPersona` when Redis is on; validates persona exists via `getPersonaMeta`.
- **Outbound:** `lib/twilio/outbound-call.ts` — Twilio **Calls REST** create. `POST /api/admin/twilio/outbound` — `Authorization: Bearer <TWILIO_OUTBOUND_API_SECRET>`, JSON `{ to, personaId? }`, requires `TWILIO_ACCOUNT_SID`, `TWILIO_AUTH_TOKEN`, `TWILIO_VOICE_FROM_NUMBER`. Uses `getPublicBaseUrl` + `/api/twilio/voice?personaId=…` as first TwiML URL.
- **Redis:** `setCallPersona`, `getCallPersona`, persona key cleared with `clearCallMessages`; `setCallMessages` refreshes persona key TTL when `personaId` option passed.
